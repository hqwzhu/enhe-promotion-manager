# Chrome 扩展指南

Chrome 扩展是 Manifest V3 的本地入口，用来把当前产品页面转换成可审阅的 Codex 命令。完整研究、生成、发布包和复盘仍由本地 Skill 完成。

## 安装方式

### Chrome Web Store

打开 [现有商店条目](https://chromewebstore.google.com/detail/enhe-promotion-manager/dloklkbnmoigemnfigbkibogmgbieppl)，点击安装。商店当前公开版本（发布前）为 `0.5.2`。

### 未打包扩展

```powershell
git clone https://github.com/hqwzhu/enhe-promotion-manager.git
cd enhe-promotion-manager
```

在 `chrome://extensions` 打开开发者模式，选择“加载已解压的扩展程序”，指向 `extension\chrome`。源码公开发行版本为 `0.5.3`。

## 当前标签页捕获

捕获必须由用户发起：

1. 打开产品网页或网站入口。
2. 点击扩展图标。
3. 点击“使用当前标签页”。
4. 核对 URL、标题、目标平台和输出目录。
5. 点击生成命令并复制到 `skill\viral-product-copy-video-generator` 目录执行。

扩展读取当前标签页的 URL 与标题作为任务输入，不会后台批量读取标签页，也不会把 Cookies、Chrome 登录配置或页面令牌上传到本公开仓库或公开发行包。

## 语言切换

界面支持中文与英文。切换只改变扩展显示文本和命令选择提示；Skill 的文件输出、平台参数和安全边界不因语言切换而改变。语言偏好保存在 Chrome 扩展本地存储。

## 命令类型

扩展可生成以下本地入口：

| 命令类型 | 用途 |
| --- | --- |
| 一键 Skill 执行 | 从产品 URL 运行完整或指定深度的本地工作流 |
| 浏览器发布会话 | 读取发布队列，准备可见入口和字段，停止在最终提交前 |
| 发布解锁包 | 汇总平台入口、凭据变量名、发布载荷和真实证据模板 |
| 爆款证据初始化 / 收件箱 | 创建公开内容研究的补采目录与导入入口 |
| 真实证据收件箱初始化 / 收件箱 | 创建并导入发布 URL、指标、评论、订单和收入文件 |
| 效果监控 | 读取公开指标或用户导出，生成表现与下一轮建议 |
| 本地平台证据 | 针对平台、关键词或详情目标生成本机证据采集命令 |
| 最终就绪审计 | 检查事实、媒体、发布、证据和下一轮状态 |
| 定时任务初始化 / 运行定时任务 | 生成 Windows 可审阅的本地调度配置和运行入口 |
| Windows 任务脚本 | 输出供用户审核后注册的任务脚本 |

这些是非支付命令。公开发行同步检查的 11 项脚本以 [版本同步说明](version-sync.md) 为准。

## 本地优先路径

扩展只生成命令和保存必要的本地界面偏好。产品读取、平台研究、媒体生成、发布包、证据和复盘都写到你选择的本机 `--out-dir`。扩展原有许可证密钥、license endpoint、usage endpoint、Hosted Run endpoint、结算页和账单入口仍在 UI 中，但 Hosted Worker 保持关闭，公开发行不把托管执行描述为可用能力。

## 最终发布与证据

浏览器辅助选项可打开平台入口、填写可见字段、保存截图和生成核对清单；它会在最终提交前停止。官方 API 路径要求用户自己的凭据、账号授权和明确批准。发布后应登记真实 URL，并通过证据收件箱导入真实指标、评论、订单和收入。

## 边界

- Cookies 与 Chrome 登录配置只留在本机，不上传到本公开仓库或公开发行包。
- 工具不会规避 CAPTCHA、平台风险控制、登录检查或账号授权。
- 最终发布需要用户审核和操作，扩展不会替代用户做最终平台确认。
- 真实 URL、指标、评论、订单和收入才是复盘证据；不会虚构数据。
- 支付、订阅、许可证、点数和账单后端仅排除在同步结论之外；扩展原有 billing UI 和 `billing-contract.json` 保留。
