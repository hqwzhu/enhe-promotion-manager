# 数据与隐私

公开发行采用本地优先的数据边界。完整隐私政策见：https://www.enhe-tech.com.cn/promotion-manager/privacy

## 数据存放位置

| 数据类型 | 默认位置 | 是否进入公开仓库/发行包 |
| --- | --- | --- |
| 产品事实、草稿、媒体、发布包、报告 | 用户指定的本机 `--out-dir`，通常为 `promotion-output` | 否 |
| 真实证据收件箱 | 用户指定的本机目录，通常为 `promotion-evidence-inbox` | 否 |
| Chrome 扩展语言和界面设置 | Chrome 扩展本地存储 | 否 |
| 扩展原有许可证/账单设置 | Chrome 扩展本地存储；只有用户主动调用配置的 ENHE 接口时才按界面提示发送 | 否 |
| Cookies 与 Chrome 登录配置 | 用户本机 Chrome profile 或 Sidecar 本机目录 | 否 |
| MediaCrawler checkout、虚拟环境、身份盐 | `%LOCALAPPDATA%\ENHE\promotion-manager\mediacrawler` 或用户指定目录 | 否 |
| Sidecar 原始采集文件 | 本机临时目录；默认归一化后清理，`--keep-raw` 时由用户管理 | 否 |

公开仓库不包含 Cookies、Chrome profile、运行态、用户输出、账号凭据、订单数据或客户数据。公开发行构建会排除 `.env`、`promotion-output`、用户数据目录和其他运行时路径。

## Hosted Worker

Hosted Worker：关闭。公开版不要求托管后端，也不把托管运行描述为可用服务。本地 Skill 不需要订阅即可运行。

扩展原有许可证、支付、订阅、点数和账单 UI 仍保留，但它们不纳入本次非支付命令与随包 Skill 的同步结论；`billing-contract.json` 继续存在。只有用户主动操作相关界面时，才应按产品页面和隐私政策核对数据传输与接口状态。

## 数据保留

- 本地输出、证据收件箱和媒体由用户控制，默认保留到用户自行删除。
- 扩展本地设置由 Chrome 管理，用户可通过扩展设置、清除浏览器站点/扩展数据或卸载扩展删除。
- Sidecar 默认清理归一化后的临时原始数据；显式 `--keep-raw` 的文件由用户决定保留周期，应尽快删除不再需要的敏感原始内容。
- 公开仓库和公开发行包不应保留任何用户 Cookies、登录配置、真实订单或收入数据。
- 用户主动提交到官方平台或 ENHE 网站的内容，适用对应平台或网站的保留政策。

## 用户应如何保护数据

- 使用环境变量或本机 `.env` 保存 API/OAuth 凭据，不要提交到 Git。
- 分享报告前检查 URL 参数、截图、评论、订单、收入和日志中的个人信息。
- 不要上传 Chrome profile、Cookies 文件、Sidecar 原始目录或许可证密钥。
- 为客户运行时，把每个客户的输出目录和证据目录分开，并只分享经审核的交付物。

## 隐私边界

- Cookies 与 Chrome 登录配置只留在本机，不上传到本公开仓库或公开发行包。
- 最终发布需要用户审核和操作；工具不会替用户完成最终平台提交。
- 工具不会规避 CAPTCHA、平台风险控制、登录检查或账号授权。
- 真实 URL、指标、评论、订单和收入才可作为真实证据；不会虚构数据。
- 支付、订阅、许可证、点数和账单后端仅排除在功能同步结论之外；扩展原有 billing UI 和 `billing-contract.json` 保留。
