# 安全政策

## 私下报告漏洞

请将安全漏洞、敏感数据暴露、权限边界错误或供应链风险私下发送至 huqingwei5942@gmail.com。建议包含：受影响版本、复现步骤、预期与实际行为、风险说明，以及不含真实秘密的最小复现材料。

请不要在公开 Issue、Discussion、Pull Request、截图、日志或演示仓库中发布 API 密钥、Cookies、会话令牌、许可证密钥、支付信息、账号数据、Chrome 登录配置、订单明细或客户个人信息。若材料含敏感内容，请先脱敏并通过邮件协调安全传递方式。

## 支持范围

当前公开源码/发行候选版本为 `0.5.4`；Chrome 商店已公开版本仍为 `0.5.3`。安全修复优先面向最新源码候选版本；请在报告中注明实际版本与安装来源。

## 安全边界

- Hosted Worker 保持关闭，公开发行不把托管执行描述为可用服务。
- Cookies 与 Chrome 登录配置只保存在本机，不得上传到本公开仓库、公开发行包或公开问题报告。
- 平台发布需要用户审核与操作；官方 API 还需要用户自己的凭据、授权和明确批准。
- 工具不会规避平台验证、登录检查或风险控制。
- 复盘只接受真实发布 URL、真实指标、真实评论、真实订单和真实收入；不得把演示数据标记为真实证据。
- 支付、订阅、许可证、点数和账单后端仅排除在同步结论之外；扩展既有 billing UI 和 `billing-contract.json` 仍保留，需要独立安全评估。

## 响应方式

收到报告后，我们会先确认是否能够复现，再评估影响、修复范围和披露时间。请在修复发布或双方约定公开时间之前保持信息私密。
